This guide takes you from zero to your first authenticated request — fetching leads from your workspace. By the end you'll have a working API key and a request you can build on.

## Prerequisites

Before you begin, you need:

- A Leadey account. Sign up at [app.leadey.ai](https://app.leadey.ai) — new workspaces start on a 60-day free trial.
- An API key from your dashboard. Generate one under **Settings → API keys**.
- A terminal with `curl`, or any HTTP client.

## Get started

<Steps>
  <Step title="Create your API key" id="create-key">
    In the Cockpit, open **Settings → API keys** and select **Create key**. Copy the key and store it somewhere safe — it's shown only once.

    Treat your key like a password. It carries the permissions of your workspace, so never commit it to source control or expose it in client-side code.
  </Step>

  <Step title="Authenticate" id="authenticate">
    Every request authenticates with your key as a Bearer token in the `Authorization` header.

    ```bash
    export LEADEY_API_KEY="sk_live_your_key_here"
    ```

    ```bash
    curl https://api.leadey.ai/v1/me \
      -H "Authorization: Bearer $LEADEY_API_KEY"
    ```

    A `200` response confirms your key is valid and returns your workspace details.
  </Step>

  <Step title="Make your first request" id="first-request">
    Fetch the most recent leads in your workspace.

    ```bash
    curl https://api.leadey.ai/v1/leads?limit=10 \
      -H "Authorization: Bearer $LEADEY_API_KEY"
    ```

    The response is a JSON list of lead records — company, contact, and the buying signals that surfaced each one. Page through results with the `limit` and `cursor` parameters.
  </Step>
</Steps>
